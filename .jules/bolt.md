## 2026-05-07 - In-Memory Role Caching for Middleware\n**Learning:** Telegram bot applications processing messages heavily leverage middleware functions for validation (e.g., rate limits, ban checks). Performing a database query for roles (like `isAdmin` or `isOwner`) on every message creates severe and unnecessary latency and load bottlenecks for the DB.\n**Action:** Apply lightweight, short-lived (e.g., 5 min) in-memory TTL caches using `Map` directly in the data access functions for static or rarely-changing attributes to drastically reduce DB load, ensuring memory management by including simple randomized garbage collection if external libraries are not permitted.
